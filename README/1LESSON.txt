1) pwa - PROGRESSIVE WEB APPLICATION  Идеально работающий сайт на медленном интернете.Другими словами, разработчику предлагаются инструменты (Service Worker, Push Notifications и др.) и указываются цели (сайт/приложение должен быть быстрым в загрузке, работать на слабом коннекте, не «лагать», работать оффлайн при необходимости). Насколько далеко продвинется по этому пути разработчик зависит только от него.
в реакте файлы serviceWorker и manifest.json отвечают за то, чтобы сделать из реакта pwa
2) Babel компилит все в React без JSX 
С JSX - const app = <h2> Hello,react</h2>
Без JSX - const app = React.createElement('h2',null,'Hello,world')
3) импортим реакт только тогда, когда используем jsx. Он нужен, чтобы бабель компилил его 
4) блоки всегда надо оборачивать во что-то (див). Потому что реакт под капотом делает  React.createElement и помешяет на страницу и ему надо знать, во что это помешять
5) основная концепция реакт - разбиение приложения на компоненты
6) компоненты - это функции по-сути, должны быть написанны с большй буквы иначе Babel воспримет как обычный елемент
7) пустой обьект нельзя во внутрь  наших реакт елементов
